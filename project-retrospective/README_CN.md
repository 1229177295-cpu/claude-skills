# 项目复盘 skill

一个用于系统性项目复盘的 Claude Code 技能，覆盖从经验反思、方法论沉淀到生成可分享 Slides 的完整流程。

适用于数据分析师、UX 研究员、产品经理，以及任何希望把项目经验转化为可复用知识的人。

---

## 它能做什么

大多数复盘止步于"哪里做好了 / 哪里做差了"。这个 skill 走得更远：

1. **结构化反思** — 通过共创式对话引导，挖掘根因，而不只是记录表面现象
2. **方法论沉淀** — 从项目中提取可复用的方法（如统计检验框架、研究流程），整理为独立的参考文档
3. **Slides 生成** — 联动 `frontend-slides` skill，把方法论文档自动转化为网页版演示文稿

---

## 工作流程

| 阶段 | 核心活动 | 输出物 |
|------|---------|--------|
| Phase 1 | 项目信息对齐 | 《项目概况卡片》 |
| Phase 2 | 各环节完成度复盘 | 《完成度评估矩阵》 |
| Phase 3 | 深度反思对话 | 《反思对话记录》 |
| Phase 4 | 系统化提炼 | 《复盘洞察报告》 |
| Phase 5 | SOP + 检查卡片生成 | 《可复用 SOP》+《核心检查卡片》 |
| Phase 6 | 方法论沉淀 | 《方法论文档》 |
| Phase 7 | Slides 生成 | 《方法论 Slides（HTML）》 |

---

## 安装方式

将 `project-retrospective/` 文件夹复制到你的 `~/.claude/skills/` 目录下，重启 Claude Code 后自动生效。

```
~/.claude/skills/
└── project-retrospective/
    ├── SKILL.md
    ├── README.md
    └── README_CN.md
```

## 触发方式

在对话中提到以下关键词时，skill 会自动激活：

**复盘相关：**
复盘、回顾、retro、项目总结、项目回顾、经验沉淀、反思、lessons learned、做完总结、优化建议、改进方向

**方法论相关：**
方法论沉淀、把方法整理成文档

**Slides 相关：**
生成 slides、做成 PPT、生成演示文稿、可以分享的材料

或者直接说：**"帮我复盘一下这个项目"**

## 设计理念

- **共创而非生成** — skill 先问再结论，你来决定输出的方向
- **根因而非症状** — 每个"坑"都追溯底层原因，不只是记录现象
- **方法而非结论** — 方法论文档沉淀的是"怎么做"，而非项目特有的结论
- **渐进深度** — 每个 Phase 都有独立价值，任意阶段停下都有完整产出
